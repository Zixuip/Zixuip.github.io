      # Echarts

      # 一、安装

      • 官网下载
      • Github拉取
      • 通过npm命令npm install echarts --save

      使用ES Module：

      import * as echarts from 'echarts';

      使用CommonJS：

      var echarts = require('echarts');
      • 引用CDN

      # 二、引用

      <!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <!-- 引入 ECharts 文件 -->
    <script src="echarts.min.js"></script>
</head>
</html>

      # 三、快速使用

      # 1. 初始化

      1. 给Echarts准备一个具有高和宽的 DOM 容器
      <div id="main" style="width: 600px;height:400px;"></div>
      1. 通过 echarts.init 方法进行初始化一个实例，并通过 setOption 方法生成
      <script type="text/javascript">
        // 基于准备好的dom，初始化echarts实例
        var myChart = echarts.init(document.getElementById('main'));
        // 指定图表的配置项和数据
        var option = {
            title: {
                text: 'ECharts 入门示例'
            },
            tooltip: {},
            legend: {
                data:['销量']
            },
            xAxis: {
                data: ["衬衫","羊毛衫","雪纺衫","裤子"]
            },
            yAxis: {},
            series: [{
                name: '销量',
                type: 'bar',
                data: [5, 20, 36, 10, 10, 20]
            }]
        };
        // 使用刚指定的配置项和数据显示图表。
        myChart.setOption(option);
    </script>

      echarts.init 相关参数：

      1. dom ： 实例容器，一般是一个具有高度的 div 元素

        特别注意 ：

        • 如果 div 是隐藏的，会获取不到 div 的高度导致初始化失败。
        • 也可以通过 style.width 和 style.height 来给 div 定值
        • 亦或是 div 初始化后，调用 echartsInstance.resize 调整大小

      2. theme ：应用的主题

      • 可以是一个主题的配置对象，也可以使用已经通过 echarts.registerTheme 注册的主题名称

      1. opts ：附加参数

        • devicePixelRatio ：设备像素比，默认取浏览器的值 window.devicePixelRatio
        • renderer ：渲染器。支持 canvas 和 svg
        • width ： 可现实指定实例宽度，单位为像素。null/undefined/auto，则表示自动取 dom（实例容器）的宽度。
        • height ：可显式指定实例高度，单位为像素。如果传入值为 null/undefined/auto，则表示自动取 dom（实例容器）的高度。

      # 四、基础概念

      # 1. echarts 实例

      一个网页可以有多个实例，每个实例可以创建多个图标和坐标系（通过option来描述）【每个实例独占一个 DOM 节点】

      # 2. 系列（series）

      • 一组数值以及他们映射成的图

      • 一个系列至少包含：

        • 一组数值

        • 图标类型（series.type）

        • 其他相关参数

      # 3. 组件（component）

      在系列之上，echarts中各种内容，被抽象为 ”组件“

      # 4. option

      描述对图标的各种需求，也可以理解为：定义了图标的 数据 、 数据如何映射成图形 、 交互行为 【参考上图】

      # 5. 组件的定位

      • [类 CSS 的绝对定位]

      多数组件和系列，都能够基于 top / right / down / left / width / height 绝对定位

      • [中心半径定位]

      只有 圆形 的组件或系列，才可以使用。并且是根据 center (中心) 和 radius (半径) 来决定位置

      例如：pie (饼图)、sunburst (旭日图)、polar (极坐标系)

      • [其他定位]-【特殊的看文档的使用说明】

      # 6. 坐标系

      直角坐标系 ：xAxis (直角坐标系X轴)、yAxis (Y轴)、grid(直角坐标系底板)
