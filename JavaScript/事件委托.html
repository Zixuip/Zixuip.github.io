# 事件委托

利用 事件冒泡 ，只指定一个 事件处理程序 ，就可以管理 某一类 的所有事件

  • 在JavaScript中，访问 dom 次数越多，引起浏览器 重绘 与 重排 的次数也越多，就会延长页面的交互 就绪时间
  • 使用事件委托，可以把多次的 dom 操作集中起来，进行一次交互触发，从提高性能

# 原理：

利用 事件冒泡 把底层的多次 dom 操作，变成由 元素父级 来接收并触发一次

例如： div>ul>li>a 将 a、li、ul 三者的事件，委托给顶层的 div 来触发，即使底部有无数个子元素，都只会由 div 触发一次

适合做 事件委托 的事件

  • click
  • mousedown
  • mouseup
  • keydown
  • keyup
  • keypress

# 例子：

  <ul id="nav">
    <li>111</li>
    <li>222</li>
    <li>333</li>
    <li>444</li>
  </ul>
  <script>
    window.onload = function () {
      var _ul = document.querySelector("#nav");
      _ul.onclick = function (e) {
        var e = e || window.event;
        var target = e.target || e.srcElement;
        if (target.nodeName.toLowerCase() == 'li') {
          alert(target.innerHTML);
        }
      }
    }
  </script>

<div id="box">
    <input type="button" id="add" value="添加" />
    <input type="button" id="remove" value="删除" />
    <input type="button" id="move" value="移动" />
    <input type="button" id="select" value="选择" />
  </div>
  <script>
    window.onload = function () {
      var Add = document.querySelector("#add");
      var Remove = document.querySelector("#remove");
      var Move = document.querySelector("#move");
      var Select = document.querySelector("#select");

      Add.onclick = function () {
        alert('添加');
      };
      Remove.onclick = function () {
        alert('删除');
      };
      Move.onclick = function () {
        alert('移动');
      };
      Select.onclick = function () {
        alert('选择');
      }

    }
  </script>

<ul id="test">
    <li>
      <p>11111111111</p>
    </li>
    <li>
      <div>
        22222222
      </div>
    </li>
    <li>
      <span>3333333333</span>
    </li>
    <li>4444444</li>
  </ul>
  <script>
    var _ul = document.querySelector('#test');
    _ul.addEventListener('click', function (e) {
      var target = e.target;
      while (target !== _ul) {
        if (target.tagName.toLowerCase() == 'li') {
          alert('li click~');
          break;
        }
        target = target.parentNode;
      }
    })
  </script>
