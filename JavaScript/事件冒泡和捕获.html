# 事件冒泡和捕获

# 1. 事件

用户或者浏览器自身的 某种行为

# 2. 事件流

  • 冒泡
  • 捕获

# 3. 事件冒泡

事件，从底层开始触发，一直传到顶层（触发底部div）

# 4. 事件捕获

从顶部，捕获到底部（触发底部div）

# 5. DOM事件流

  1. 事件 捕获 阶段
  2. 处于 目标 阶段
  3. 事件 冒泡 阶段

# 6. 事件处理程序

响应某个事件的 函数

# 6.1. HTML

会触发 事件冒泡

缺点：

  • 存在时间差
  • HTML和JavaScript代码紧密耦合，如果要更换事件处理程序，就必须改动HTML代码和JavaScript代码。

# 6.2. DOM0级

会触发 事件冒泡

作用域： 局部 的 handler

注意：每个事件，只支持 一个事件处理程序

btn.onClick = () => {
  console.log('我是DOM0级事件处理程序');
}
btn.onClick = null;

缺点：

  • 不能给一个元素同时添加两个事件
  • 不能控制元素的事件流（捕获或冒泡）

# 6.3. DOM2级

默认(false)使用 事件冒泡 ；当第三个参数：true时，使用 事件捕获

添加多个事件时，会按照 添加顺序执行

// addEventListener(事件类型(如"click"),处理函数，true/false);---添加事件侦听器
// removeEventListener()   ---删除事件侦听器
btn.addEventListener('click', () => {
  console.log('我是DOM2级事件处理程序');
}, false);
btn.removeEventListener('click', handler, false)

# 6.4. IE事件处理程序

IE8及更早版本，只支持 事件冒泡

作用域： 全局 ，即 this==window

触发顺序：当 添加多个事件时 ，按照添加顺序的 反方向 来执行

//接收两个参数
//1. 事件处理程序名称。如onclick、onmouseover
//2. 事件处理程序函数。如function(){alert("clicked");}
attachEvent()
detachEvent()

# 6.5. 跨浏览器的事件处理程序

注意 ：先判断 DOM2级 -> 再判断 IE事件 -> 最后判断 DOM0级

1.创建一个方法addHandler():区分使用哪种方法来添加事件； 2.创建一个对象EventUtil。拥有两个方法。 3.addHandler(要操作的元素，事件名称，事件处理函数)。 4.removeHandler(要操作的元素，事件名称，事件处理函数)。

  • DOM中的事件对象（ event ）
  1. type —— 事件类型
  2. target —— 事件目标
  3. stopPropagation() —— 阻止事件冒泡
  4. preventDefault() —— 阻止事件的默认行为
  • IE中的事件对象（ event 或者 window.event ）
  1. type —— 事件类型
  2. srcElement —— 事件目标
  3. cancelBubble=true —— 阻止事件冒泡
  4. returnValue=false —— 阻止事件的默认行为
// 接收三个参数
// 要操作的元素——通过dom方式获取
// 事件名称--不是处理方法，所以不需要on，如click
// 事件处理函数
addHandler()  --这个方法职责是视情况来使用DOM0级、DOM2级、IE事件处理程序来添加事件。
removeHandler()--这个方法就是移除使用addHandler添加的事件。
