    # JavaScript高级程序设计

    # 3、基本概念

    # 3.1. 标识符

    第一个字符必须是：一个字母、下划线( _ ) 或 一个美元符号 ( $ )

    其他字符可以是字母、下划线、美元符号或数字

    一般采用 驼峰大小 格式来——首字母小写，其他每个单词首字母大写

    # 3.2. 数据类型

    五种简单数据类型：Undefined 、Null 、Boolean 、Number 、String

    一种复杂数据类型：Object ——本质由 一组无序的名值对组成的

    # 3.2.1. typeof操作符

    检测变量的数据类型

    类型 结果
    Undefined “undefined”
    Null “object”
    Boolean “boolean”
    Number “number”
    BigInt “bigint”
    String “string”
    Symbol（ES2015新增） “symbol”
    宿主对象（由JS环境提供） 取决于具体实现
    Function对象 “function”
    其他任何对象 “object”

    # 3.2.2. Undefined类型

    var声明变量，未初始化并调用，报undefined

    let 和 const 声明变量，未初始化并调用，提示错误信息

    # 3.2.3. Null类型

    # 3.2.4. Boolean类型

    返回值 true 或 false

    # 3.2.5. Number类型

    整数 和 浮点数值（也被称为双精度数值）

    1. 浮点数值

    数值中必须包含一个小数点，并且小数点后面必须至少有一位数字

    • 最高精度是 17 位小数
    1. 数值范围
    • Number.MIN_VALUE：5e-324
    • Number.MAX_VALUE：1.7976931348623157e+308

    若超过了范围，会自动变成 Infinity 值，或者 -Infinity

    1. NaN

    非数值（Not a Number）：表示一个本来要返回数值的操作数未返回数值的情况

    NaN和任何数值都 不相等 ，包括 本身

    • isNaN ：
      • 帮我们确定某个参数是否 "不是数值" 【 true 表示不是数值，反之，是数值
      • 基于对象使用 isNaN：首先调用对象的 valueOf() ，返回值不能转为数值，再调用 toString() 方法，在进行检测

    # 3.2.6. 数值转换

    把非数值转为数值：Number() 、parseInt() 和 parseFloat()

    • Number()转换规则

    1. Boolean值：true为1、false为0

    2. 数字值：简单的传入和返回

    3. null：返回0

    4. undefined：返回NaN

    5. 字符串：

      • 只包含数字：转为十进制数值，并返回（前导的0会被忽略：即“01“，返回1）
      • 包含有效浮点格式：如“1.1”，则返回对应的浮点数值（同样会忽略前导0）
      • 空字符串：返回0
      • 字符串中包含上述之外的字符，返回NaN

    6. 对象：

      • 先调用对象的 valueOf() ，按照上面规则转换和返回相应的值，如果返回结果是NaN，再调用 toString() 方法，再次根据上述规则，返回相应的字符串值
    • parseInt()
    1. 忽略字符串前面的空格
    2. 找到第一个非空格字符，就返回
    3. 如果第一个字符不是数字或负号：返回NaN
    4. 例子：“1234blue”，转换为 1234；“22.5”，转换为 22。
    5. 能识别（十进制、八进制和十六进制数）
    • parseFloat()
    1. 解析遇到第一个 无效的浮点数字字符，就结束
    2. 只能解析 十进制
    3. 若要解析的字符串没有小数点，或小数点后面全是零的，返回 整数
    4. 例子：“22.34.5”，转换为 22.34

    # 3.2.7. String类型

    由零或多个16位Unicode字符组成的字符序列，可以用（"）或（‘）表示

    • 转义序列

      • \n ：换行
      • \t ：制表
      • \b ：空格
      • \r ：回车
      • \f ：进纸
      • \\ ：斜杠
      • \' ：单引号，例：'He said , \ 'hey .\ ' '
      • \" ：双引号，例：”He said , \ “hey .\ “ ”
      • \xnn ：以十六进制代码nn表示的一个字符（其中n为0~F）例：\x41->“A”
      • \unnnn ：以十六进制代码nnnn表示的一个Unicode字符（其中n为0~F）

    • 转换为字符串

      • 如果值有 toString()方法，调用（没有参数）并返回十进制结果
      • null，返回“null”
      • undefined，返回“undefined”

    数值、布尔值、对象和字符串值都有一个toString()方法，但 null和undefined没有

    默认情况下：toString()以 十进制 格式进行返回

    可以给方法添加参数，参数代表二进制、八进制、十六进制等，乃至其他有进制的格式

    # 11、DOM扩展

    # 11.1. Selectors API

    核心：querySelector() 和 querySelectorAll()

    # 11.1.1. querySelector()

    1. 接收一个 CSS选择符 ，返回匹配到的第一个元素，若匹配失败则返回null
    2. 通过 Document 调用方法时，会在 文档元素 的范围内查找匹配元素
    3. 通过 Element 调用方法时，只会在该元素后代元素范围内匹配元素

    # 11.1.2. querySelectorAll()

    1. 接收参数和 querySelector 一致
    2. 该方法返回的是 所有匹配 的元素，即：一个 NodeList 实例(并且会带上元素的属性和方法)

    # 11.1.3. matchesSelector()

    1. 接收一个 css 选择符 ，调用元素跟选择符进行匹配，如何返回 true 否则返回 false

    # 11.2. 元素遍历

    1. childElementCount：返回子元素（不包括文本节点和注释）的个数；
    2. firstElementChild：指向第一个子元素；
    3. lastElementChild：指向最后一个子元素；
    4. previousElementSibling：指向前一个同辈元素；
    5. nextElementSibling：指向后一个同辈元素；

    # 11.3. HTML5

    # 11.3.1. getElementsByClassName()

    1. 接收一个参数—— 包含一或多个类名的字符串 ，返回 NodeList （指定类的所有元素），注意 ：只有位于调用元素子树中的元素才返回
    2. document 对象调用时，始终返回匹配成功的所有元素，
    3. 在 元素 上调用时，只会返回 后代元素 中匹配的元素
    • classList属性

    操作类名时，可以通过该属性进行 添加 、删除 、和 替换 类名

    1. add(value)：将给定的 字符串值 添加到列表中；若存在，则不添加
    2. contains(value)：列表中是否存在给定的值，存在返回 true ，否则返回 false
    3. remove(value)：从列表中删除给定的字符
    4. toggle(value)：如果列表中已经存在给定的值，删除它；如果没有，则添加它

    # 11.3.2. 焦点管理

    document.activeElement属性：始终会引用DOM中当前获得了 焦点 的元素

    # 11.3.3. HTMLDocument的变化

    1. readyState
    • loading：正在加载文档
    • complete：已经加载完文档
    • 使用场景：实现一个指示文档已经加载完成的指示器
    1. compatMode（兼容模式）

    为了告诉开发人员浏览器采用了哪种渲染模式

    • 标准模式：document.compatMode 的值等于 CSS1Compat
    • 混杂模式：document.compatMode 的值等于 BackCompat
    1. head

    引用文档的 <head> 元素

    # 11.3.4. 字符集属性

    1. charset

    表示文档中实际使用的字符集，也可以用来指定新字符集

    1. defaultCharset

    表示根据默认浏览器及操作系统的设置，当前文档默认的 字符集 应该是什么

    若没有默认 字符集 ，1、和2、的属性值可能会不一样

    # 11.3.5. 自定义数据属性

    1. 添加 非标准的 属性

    元素前缀： data- (目的：是为了元素提供与渲染无关的信息，或者提供语义信息)

    1. 添加自定义属性 后

    可以用过 dataset ：访问自定义属性的值

    dataset 属性是 DOMStringMap 的一个实例，也就是一个映射

    # 11.3.6. 插入标记

    1. innerHTML

    读模式：返回值为：与调用元素的所有子节点（包括元素、注释和文本节点）对应的HTML标记

    写模式：会根据指定的值，创建新的DOM树。（会 完全 替换调用元素原先的所有子节点）

    • 不支持该属性的元素有：<col>、<colgroup>、<frameset>、<head>、<html>、<style>、<table>、<tbody>、<thead>、<tfoot> 和 <tr>
    • IE8：window.toStaticHTML() 方法
      • 接受一个参数，一个HTML字符串
      • 返回一个：经过处理后的版本——从源HTML中删除所有脚本节点和事件处理程序属性
    1. outerHTML

    读模式：返回——调用它的元素及所有子节点的HTML标签

    写模式：根据指定的HTML字符串 创建 新的 DOM子树 ，再完全替换元素

    1. insertAdjacentHTML

    接收两个 参数 ： 插入位置 和 要插入的 HTML文本

    第一个参数必须是下列值之一（必须小写）：

    • beforebegin：在当前元素 之前 插入 一个紧邻的 同辈 元素
    • afterbegin：在当前当前元素 之下 插入 一个新的子元素 或 在 第一个子元素 之前再插入新的子元素
    • beforeend：在当前元素 之下 插入 一个新的子元素 或 在 最后一个子元素 之后 再插入新的元素

    # 11.3.7. scrollIntoView()

    # 11.3.8. contains()

    接收一个参数：要检测的后代节点

    返回值：如果被检测的节点是后代节点，返回 true ，否则返回 false

    • compareDocumentPosition() ：也可以确定节点间的关系
    掩码 节点关系
    1 无关（给定的节点不在当前文档中）
    2 居前（给定的节点在DOM树中位于参考节点之前）
    4 居后（给定的节点在DOM树中位于参考节点之后）
    8 包含（给定节点是参考节点的祖先）
    16 被包含（给定的节点是参考节点的后代）

    # 11.3.9. 插入本文

    1. innerText

    一、可以操作元素中包含的所有文本内容，包括子文档树的文本

    二、读取值时：按照 由浅入深 的顺序，将 子文档树 中的所有文本拼接起来

    三、写入值时：删除元素的所有子节点，插入包含相应文本值的文本节点

    1. outerTxt

    作用范围扩大到了包含调用它的节点，其余用法跟 innerText 基本没区别

    读取文本值时：和 innerText 结果一致

    写入模式：不只是替换调用它的元素的子节点，而是替换整个元素（包括子节点）

    # 11.4. 滚动

    1. scrollIntoViewIfNeed(alignCenter)：只在 当前元素 在视口中不可见的情况下，才滚动浏览器窗口或容器元素，最终让它可一件。如果当前元素在视口 可见，这个方法什么都不做
    2. scrollByLines(lineCount)：将元素的内容滚动指定的行高，lineCount值可以是正值，也可以是负值
    3. scrollByPages(pageCount)：将元素的内容滚动指定的页面高度，具体高度由元素的高度决定
